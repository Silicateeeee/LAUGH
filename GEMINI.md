# MASTER PROMPT

## Building
To build any projects, always use `distrobox`. You can enter a fedora container using `distrobox enter cpp`.

## Dependencies
Whenever you come across a dependency that is not yet installed, or needs to be updated, use `sudo dnf` with fedora package naming.

## Running
This project interacts with other programs' memory. Always run with `sudo` to ensure it runs as intended.

## Errors
When you come across an error, or accidentally create an error, increment this counter: `Errors: 0`.
Do not go haywire and modify the entire project or nuke files. Only do what is absolutely necessary to resolve errors.

## Warnings
For the most part, ignore warnings unless they might raise some actual issues.
